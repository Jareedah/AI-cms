    The post Capture UTM parameters with Attributer first appeared on Perfex CRM.]]>     Attributer is a tool that can pass marketing attribution data (like the fact a lead came from Paid Search, Organic Search, Paid Social, etc) into Perfex.

    In this article, we will show you how to set up Attributer on your website and get the data into Perfex.

    What is Attributer?

    Attributer is a small piece of code that you place on your company’s website.

    When a visitor lands on your site, Attributer looks at a bunch of technical information (same things as tools like Google Analytics look at) to determine where the visitor came from.

    It then categorizes the visitor into a standardized set of marketing channels (the same channels you see in tools like Google Analytics, such as Organic Search, Paid Social, Paid Search, etc) and saves the data in the user’s browser.

    Then when the visitor completes a form on your website, Attributer writes the attribution data into hidden fields in your forms and it is captured by your form tool.

    Finally, you can then send this data into Perfex where you can view it on the Lead record, use it to run reports, etc. 

    Setup Instructions

    Follow these step-by-step instructions below to start using Attributer with Perfex.

    Step 1: Add the Attributer code to your website

    When you start a 14-day free trial, you will be given a snippet of code to add to your website. The process for adding this code differs depending on what CMS system you are using, but full instructions (with screenshots) can be seen here:

    • WordPress
    • Webflow
    • Squarespace
    • Wix
    • Other

    Step 2: Add hidden fields to your forms

    The next thing you need to do is add a series of hidden fields to your forms. This will give Attributer a place to write the UTM parameters and other lead source data into.

    Attributer works with 50+ different form builders and the process is slightly different for all of them, but generally speaking, you just need to add 6x hidden fields to your forms and then give them a specific name and default value.

    The fields you need to add are as follows:

    Hidden Field #1

    • Name = Channel
    • Default Value = [channel]

    Hidden Field #2

    • Name = Channel Drilldown 1
    • Default Value = [channeldrilldown1]

    Hidden Field #3

    • Name = Channel Drilldown 2
    • Default Value = [channeldrilldown2]

    Hidden Field #4

    • Name = Channel Drilldown 3
    • Default Value = [channeldrilldown3]

    Hidden Field #5

    • Name = Landing Page
    • Default Value = [landingpage]

    Hidden Field #6

    • Name = Landing Page Group
    • Default Value = [landingpagegroup]

    You ultimately want your form to look a bit like this:

    Step 3: Send the data into Perfex CRM

    Now that you have added the Attributer code to your website and added the hidden fields to your forms, the UTM parameters and other attribution should be getting captured by your form tool.

    You can check this by looking at the Submissions. They should look a bit like this:

    If that’s the case, then the final step is to send the data from your form tool to Perfex.

    Suppose you already have an integration set up (via Zapier or Perfex’s email integration). In that case, you just need to update the field mappings to send the additional data captured by your form tool into Perfex.

    How to use this data

    Now that you have attribution data on each new lead coming into Perfex, let’s look at different  ways you can use this data to help grow your business:

    Tailor your lead follow-up

    When you have information about what channel the lead came from, what ad they clicked, and what pages they landed on, you can use it to send more targeted follow-up messages.

    For example, if we had a lead from Organic Search, their landing page was our homepage, and they submitted a demo form request, we could probably assume they were interested in Perfex and in a later stage of the buying cycle. As a result, the messaging in our emails should probably be focused on getting the demo set up, educating them on the features, etc.

    Alternatively, if we saw someone came from a Facebook Ad, landed on our blog post about ‘10 ways to grow your business’ and became a lead by downloading our eBook on business growth, we could probably assume they are not familiar with Perfex or even looking for a CRM, so the messaging should be more focused on how using a CRM can help them grow.

    Report on the channels driving leads, customers, and revenue

    When you have the data about where each lead came from in Perfex, you can run reports showing which channels, campaigns, etc are driving your leads.

    For instance, you could run a report that shows the number of leads you got by channel.

    This would allow you to see which channels (Paid Search, Paid Social, Organic Search, etc) are generating you the most leads (and subsequently where you should be focusing your efforts to get more).

    The post Configure email with Microsoft Azure and OAUTH 2 first appeared on Perfex CRM.]]>     Starting from version 3.0.5, you can configure a general Microsoft email using OAuth2, to do this on your Perfex CRM installation follow the steps below, note that you will need to create an Azure application and authorize it in the Perfex CRM email configuration section at Setup -> Settings-> Email

    Microsoft Azure client secrets expires after a maximum of between 3 and 24 months (you can select the length yourself).

    If your authentication suddenly doesn’t work anymore, please check the if the Client Secret has expired.

    Notes

    • New App Registration Requirement: As of the update, personal Microsoft accounts (e.g., @outlook.com, @hotmail.com) cannot register apps outside of a directory. Ensure you are working within an AAD tenant.
    • OAuth Consent: Users may need to authorize your app the first time they use it.

    For more detailed information, refer to Microsoft Documentation.

    Steps

    1. Register an Application in Azure Active Directory

    • Log in to the Azure Portal: https://portal.azure.com.
    • Navigate to Azure Active Directory > App Registrations.
    • Click + New Registration.
    • Provide:
      • Name: Enter a recognizable name (e.g., Perfex CRM Email Integration).
      • Select who should be able to use this application:
      • Supported Account Types: Select who should be able to use this application:
        • Accounts in this organizational directory only (Single tenant) for internal use.
        • Accounts in any organizational directory (Multi-tenant) if you want to support other AAD tenants.
        • If you are not sure about this option, select: Accounts in any organizational directory (Any Microsoft Entra ID tenant – Multitenant) and personal Microsoft accounts (e.g. Skype, Xbox)
      • Redirect URI: Enter https://perfex-crm-url.com/admin/smtp_oauth_microsoft/token.
      • Ensure to replace perfex-crm-url.com with your Perfex CRM installation.
      • Ensure Web is select in the redirect URI dropdown.
    • Click Register.

    2. Configure API Permissions

    • Go to API Permissions in your app’s settings (Manage -> Certificates & secrets).
    • Click + Add a permission.
    • Select Microsoft Graph > Delegated permissions.
    • Add the following permissions:
      • offline_access
      • SMTP.Send
    • (Optional) If admin consent is required, click Grant admin consent and confirm.

    3. Set Up Secret

    • In your app, go to Certificates & secrets (Manage -> Certificates & secrets).
    • Click + New client secret.
    • Add a description (e.g., Perfex CRM Secret) and choose an expiration period.
    • Click Add and copy the Value. This is your Client Secret.

    4. Note Key Details

    Save the following information for Perfex CRM configuration:

    • Application (client) ID: Found on the app’s Overview page.
    • Directory (tenant) ID (if applicable): Found on the app’s Overview page.
    • Client Secret: Copied from the previous step.

    5. Enable Email Sending in Perfex CRM

    • Log in to Perfex CRM.
    • Navigate to Setup -> Settings -> Email.
    • Select Microsoft Azure XOAUTH2 as the email protocol.
    • Ensure Mail Engine is set to PHPMailer. 
    • Ensure the Email Protocol is set to Microsoft OAuth 2.0.
    • Enter the following:
      • Client ID: Your Application (client) ID.
      • Client Secret: Your Client Secret.
      • Tenant ID: Your Directory (tenant) ID (if applicable).
      • SMTP Host: smtp.office365.com.
      • SMTP Port: 587.
      • Encryption: TLS.
      • Email: The e-mail address you will use to send emails from Perfex CRM, usually this is the e-mail address associated with the Azure application.
